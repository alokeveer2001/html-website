HTML Forms

  • Why HTML form?

    HTML forms are essential for collecting user input on web pages. They allow users to enter data, which can then be sent to a server for processing. Forms are commonly used for various purposes, such as user registration, login, feedback submission, and search functionality.

    Forms enhance user interaction and enable dynamic content generation based on user input.

  • The <input> Element: The Box for Your Stuff

    The <input> element is a versatile HTML tag used to create various types of input fields within a form. It allows users to enter data, such as text, passwords, email addresses, and more.

    The <input> element can be customized using the "type" attribute to specify the type of input field, such as "text", "password", "email", "checkbox", "radio", and others.

    Example:

    Code : 

    <form>
    <input type = "text">
</form>

    Output :

    In this example, an input field of type "text" is created within a form, allowing users to enter text data. The <input> element is self-closing and does not require a closing tag.

  • The Need for Meaning - The <label>

    The <label> element in HTML is used to define a label for an <input> element. It provides a descriptive text that helps users understand the purpose of the input field.

    Using <label> elements improves accessibility and usability, as it allows users to click on the label to focus on the associated input field.

    Example:

    Code : 

    <form>
    <label for="username">Username:</label>
    <input type="text" id="username" name="username">
</form>

    Output :

    In this example, the <label> element is associated with the <input> element using the "for" attribute, which matches the "id" of the input field. This association enhances the user experience by providing clear context for the input field.

  • The Need for Connection - id and for

    The "id" attribute in an <input> element is used to uniquely identify the input field within the HTML document. It allows for easy reference and manipulation of the input field using JavaScript or CSS.

    The "for" attribute in a <label> element is used to associate the label with a specific input field by matching the value of the "for" attribute with the "id" of the input field.

    This connection enhances accessibility, as clicking on the label will focus the corresponding input field.

    Example:

    Code : 

    <form>
    <label for="email">Email:</label>
    <input type="email" id="email" name="email">
</form>

    Output :

    In this example, the <label> element is associated with the <input> element using the "for" attribute, which matches the "id" of the input field. This association improves the user experience by providing clear context for the input field.

  • The Need for Submission - The <form> and submit

    The <form> element in HTML is used to create a form for collecting user input. It serves as a container for various input elements, such as text fields, checkboxes, radio buttons, and submit buttons.

    To submit the form, a submit button is typically included using the <input> element with the "type" attribute set to "submit".

    Example:

    Code : 

    <form>
    <label for="firstName">First Name:</label>
    <input type="text" id="firstName">
    <br><br>
    <label for="lastName">Last Name:</label>
    <input type="text" id="lastName">
    <br><br>
    <input type="submit">
</form>

    Output :





  • The Need for Data Identification - The name Attribute

    The "name" attribute in an <input> element is used to specify a name for the input field. This name is important because it is used as the key when the form data is submitted to the server.

    When a form is submitted, the values entered in the input fields are sent as key-value pairs, where the key is the value of the "name" attribute and the value is the user-entered data.

    Example:

    Code : 

    <form>
    <label for="firstName">First Name:</label>
    <input type="text" id="firstName" name="firstName">
    <br><br>
    <label for="lastName">Last Name:</label>
    <input type="text" id="lastName" name="lastName>
    <br><br>
    <input type="submit">
</form>

    Output :





    In this example, the "name" attributes for the input fields are set to "firstName" and "lastName". When the form is submitted, the data will be sent as key-value pairs, such as "firstName=John" and "lastName=Doe", where "John" and "Doe" are the values entered by the user.

  • The Need for "Select One" - Radio Buttons

    Radio buttons are a type of input element in HTML that allows users to select one option from a predefined set of choices. They are typically used when only one selection is allowed within a group of options.

    Radio buttons are created using the <input> element with the "type" attribute set to "radio". To group radio buttons together, they should share the same "name" attribute.

    Example:

    Code : 

    <form>
    <label>T-Shirt Size:</label>
    <br>
    <!-- All three are part of the "shirtSize" group -->
    <input type="radio" id="sizeS" name="shirtSize" value="small">
    <label for="sizeS">Small</label>
    <br>
    <input type="radio" id="sizeM" name="shirtSize" value="medium">
    <label for="sizeM">Medium</label>
    <br>
    <input type="radio" id="sizeL" name="shirtSize" value="large">
    <label for="sizeL">Large</label>
    <br><br>
</form>

    Output :






  • The Need for "Select Many" - Checkboxes

    Checkboxes are a type of input element in HTML that allows users to select one or more options from a predefined set of choices. They are typically used when multiple selections are allowed within a group of options.

    Checkboxes are created using the <input> element with the "type" attribute set to "checkbox". Each checkbox can have its own "name" attribute, allowing for independent selection.

    Example:

    Code : 

    <form>
    <label>Select your hobbies:</label>
    <br>
    <input type="checkbox" id="hobbyReading" name="hobbies" value="reading">
    <label for="hobbyReading">Reading</label>
    <br>
    <input type="checkbox" id="hobbyTraveling" name="hobbies" value="traveling">
    <label for="hobbyTraveling">Traveling</label>
    <br>
    <input type="checkbox" id="hobbyCooking" name="hobbies" value="cooking">
    <label for="hobbyCooking">Cooking</label>
    <br><br>
</form>

    Output :






  • A Better Button - The <button> Element

    The <button> element in HTML is used to create clickable buttons. It provides more flexibility and options compared to the <input type="submit"> element, allowing for custom content and styling.

    The <button> element can contain text, images, or other HTML elements, making it more versatile for creating interactive buttons.

    Example:

    Code : 

    <form>
    <button type="submit">Submit Form</button>
</form>

    Output :

    In this example, a button is created using the <button> element with the "type" attribute set to "submit". When clicked, it will submit the form. The <button> element allows for more customization and can include various types of content.

  • The Need for Long-Form Text (<textarea>)

    The <textarea> element in HTML is used to create a multi-line text input field. It allows users to enter longer pieces of text, such as comments, feedback, or messages.

    The <textarea> element can be customized using attributes like "rows" and "cols" to specify the visible size of the text area.

    Example:

    Code : 

    <form>
    <label for="comments">Comments:</label>
    <br>
    <textarea id="comments" name="comments" rows="4" cols="10"></textarea>
</form>

    Output :


    In this example, a multi-line text area is created using the <textarea> element. Users can enter longer pieces of text within the specified number of rows and columns.

  • The Need for Many Options - The Dropdown (<select>)

    The <select> element in HTML is used to create a dropdown list, allowing users to choose one option from a predefined set of choices. It is commonly used in forms for selecting options such as countries, states, or categories.

    The <select> element contains multiple <option> elements, each representing a selectable item in the dropdown list.

    Example:

    Code : 

    <form>
    <label for="country">Country:</label>
    <br>
    <select id="country" name="country">
        <option value="">--Please choose an option--</option>
        <option value="in">India</option>
        <option value="us">USA</option>
        <option value="uk>United Kingdom</option>
        <option value="au">Australia</option>
    </select>
</form>

    Output :




  • More Specialized Inputs (HTML5 Power-ups)

    HTML5 introduced several specialized input types that enhance the functionality and user experience of forms. These input types provide built-in validation and user-friendly interfaces for specific types of data entry.

    Some of the specialized input types include:

    • email: Validates that the input is a properly formatted email address.
    • url: Validates that the input is a properly formatted URL.
    • tel: Provides an input field optimized for telephone numbers.
    • number: Allows only numeric input and provides up/down arrows for incrementing/decrementing values.
    • date: Provides a date picker for selecting dates.
    • color: Provides a color picker for selecting colors.
